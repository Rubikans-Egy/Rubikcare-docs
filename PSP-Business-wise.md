
# 📋 **الوثيقة النهائية لمنطق أعمال Pharma Portal - PSP System**

## 🎯 **نظرة عامة**

### **الهدف:**
منصة تمكن **شركات الأدوية**
 من إنشاء وإدارة برامج دعم المرضى (PSP) التي تقدم من خلال **العيادات والصيدليات** المنضمة للبرنامج.

### **القيمة:**
- **شركات الأدوية:** توسيع انتشار أدويتها، تحسين الالتزام بالعلاج
- **العيادات:** جذب مرضى جدد، توفير علاج أفضل بتكلفة أقل  
- **الصيدليات:** زيادة المبيعات، بناء علاقات مع المرضى
- **المرضى:** الحصول على أدوية بأسعار مدعومة، متابعة صحية أفضل

## 🏢 **الهيكل المؤسسي**

### **1. المنظمات (Organizations) - الوحدة الأساسية**
كل خدمة تقدم عبر **منظمة**، وليس أفراداً:
- **👨‍⚕️ العيادات (Clinics):** تقديم التشخيص، الدعوة للمرضى
- **💊 الصيدليات (Pharmacies):** صرف الأدوية، التوصيل
- **🏢 شركات الأدوية (Pharma):** تمويل البرامج، الدعوة للعيادات والصيدليات

### **2. المستخدمون (UserProfiles) - تابعون للمنظمات**
- طبيب → تابع لعيادة
- صيدلي → تابع لصيدلية  
- موظف → تابع لشركة أدوية

### **3. النظام يولد منظمات افتراضية تلقائياً**
```
👨‍⚕️ طبيب يسجل ← "عيادة د. أحمد محمد" (تولد تلقائياً)
💊 صيدلي يسجل ← "صيدلية الصحة" (تولد تلقائياً)
```

## 🔄 **سير العمل الكامل**

### **🏗️ المرحلة 1: الإعداد التأسيسي**
```
1. شركة أدوية تسجل في النظام (منظمة Pharma)
2. النظام ينشئ خدمة افتراضية "إنشاء عيادة/صيدلية"
3. عند تسجيل طبيب/صيدلي: 
   - ينشئ حساب شخصي (UserProfile)
   - ينشئ منظمته الافتراضية (Organization)
   - تربط الخدمة المجانية بالمنظمة
```

### **🎪 المرحلة 2: إنشاء برنامج PSP**
```
1. شركة أدوية → تنشئ برنامج PSP جديد
2. تحدد: الأدوية، نسبة الدعم، التخصصات، الميزانية
3. تربط البرنامج بـ Service Product (تسعير)
```

### **📨 المرحلة 3: دعوة المنظمات**
```
1. مندوب شركة أدوية → يدعو عيادات وصيدليات للبرنامج
2. العيادة/الصيدلية → تستلم الدعوة → توافق/ترفض
3. إذا وافقت → تصبح عضو في البرنامج (PSPParticipation)
```

### **👨‍⚕️ المرحلة 4: الطبيب والعيادة**
```
1. طبيب (في عيادة منضمة) → يعرض برنامج PSP لمرضاه
2. مريض → يوافق على الانضمام
3. العيادة (Organization) → توافق على انضمام المريض
4. المريض → يصبح مسجلاً في البرنامج (PSPPatient)
```

### **💊 المرحلة 5: الصرف والدواء**
```
1. مريض مسجل → يبحث عن صيدليات منضمة قريبة
2. يختار صيدلية → يحصل على OTP/Token
3. يذهب للصيدلية → يعطي OTP
4. صيدلي → يتحقق → يصرف الدواء المدعوم
5. الصيدلية → تطلب دواء من الشركة عبر النظام (إن لزم)
```

### **📊 المرحلة 6: المتابعة والتقارير**
```
1. شركة الأدوية ← ترى:
   - عدد العيادات/الصيدليات المنضمة
   - عدد المرضى المستفيدين
   - الأدوية المصروفة
   - الميزانية المستخدمة

2. العيادة ← ترى:
   - مرضى العيادة في البرنامج
   - أدويتهم المصروفة
   - مدخراتهم المالية

3. المريض ← يرى:
   - مدخراته
   - الصيدليات القريبة
   - مواعيد الصرف القادمة
```

## 🗄️ **هيكل قاعدة البيانات**

### **✅ الجداول الحالية (معدلة):**
#### **1. PSPPrograms** - برامج الدعم
```sql
- Category (Access/Adherence/Convenience)
- SubType (خصم/تقسيط/توصيل/إلخ)
- Description, LaunchReason (جديد)
- Budget, PharmaCompanyID, ServiceProductID
```

#### **2. PSPProgramMedications** - أدوية البرنامج
```sql
- SupportType (نسبة/مبلغ)
- SupportValue (80% أو 150 ريال)
- MaxQuantityPerPatient (الحد الأقصى)
```

#### **3. PSPProgramSpecialities** - تخصصات مستهدفة (جديد)
```sql
- برنامج ←→ تخصص (Many-to-Many)
```

#### **4. PSPEnrollments → PSPParticipations** (معدل)
```sql
- ParticipantOrganizationID (بدلاً من DoctorID) ⭐
- Status, JoinDate
- InvitationID (رابط للدعوة)
```

#### **5. PSPPatients** - مرضى البرامج (معدل)
```sql
- ApprovedByClinicID (العيادة التي وافقت) ⭐
- InvitedByOrganizationID (من دعاه) ⭐
- InvitedByUserID (الطبيب المحدد)
```

### **➕ الجداول الجديدة:**

#### **6. PSPInvitations** (جديد)
```sql
InvitationID
ProgramID
InvitedOrganizationID (العيادة/الصيدلية المدعوة)
InvitedByUserID (مندوب الشركة)
Status (PENDING/ACCEPTED/REJECTED)
SourceType (SALES_REP/SYSTEM/REFERRAL)
```

#### **7. ServiceProducts** (موجود، نربطه)
```sql
- خدمة "إنشاء عيادة افتراضية" (مجانية)
- خدمة "إنشاء صيدلية افتراضية" (مجانية)
- خدمات PSP (مدفوعة: PSP-BASIC, PSP-PREMIUM)
```

## 👥 **أدوار المستخدمين**

### **1. Pharma Admin (شركة أدوية)**
**الصلاحيات:**
- إنشاء/تعديل/حذف برامج PSP
- دعوة عيادات وصيدليات
- عرض تقارير البرنامج
- إدارة الميزانية

### **2. Clinic Admin (مدير عيادة)**
**الصلاحيات:**
- قبول/رفض دعوات PSP
- دعوة مرضى للانضمام
- مراجعة طلبات انضمام المرضى
- متابعة مرضى العيادة في البرامج

### **3. Doctor (طبيب في عيادة)**
**الصلاحيات:**
- عرض برامج PSP المتاحة
- دعوة مرضى محددين
- كتابة وصفات PSP
- متابعة مرضاه في البرامج

### **4. Pharmacy Admin (مدير صيدلية)**
**الصلاحيات:**
- قبول/رفض دعوات PSP
- إدارة صرف الأدوية المدعومة
- طلب أدوية من الشركات
- متابعة المخزون

### **5. Pharmacist (صيدلي في صيدلية)**
**الصلاحيات:**
- التحقق من OTP/Token
- صرف الأدوية المدعومة
- تسجيل عمليات الصرف

### **6. Patient (مريض)**
**الصلاحيات:**
- البحث عن برامج PSP
- التسجيل في البرامج
- البحث عن صيدليات قريبة
- عرض مدخراته ومواعيده

## 💰 **النموذج الاقتصادي**

### **1. خدمات مجانية:**
- إنشاء عيادة افتراضية
- إنشاء صيدلية افتراضية  
- التسجيل الأساسي

### **2. خدمات مدفوعة (PSP Programs):**
- **PSP-BASIC:** خدمات أساسية (سعر منخفض)
- **PSP-PREMIUM:** خدمات متقدمة (سعر أعلى)
- **فواتير شهرية** لشركات الأدوية

### **3. القيمة للجميع:**
- **شركات الأدوية:** وصول أوسع ← مبيعات أعلى
- **العيادات:** مرضى جدد ← دخل أعلى
- **الصيدليات:** مبيعات أدوية ← ربح أعلى
- **المرضى:** أدوية أرخص ← توفير مالي
- **المنصة:** اشتراكات PSP ← إيرادات مستدامة

# القيمة الاستراتيجية المضافة

## 🎯 للمرضى

- **دعم شامل 360°:** من الوصول المالي إلى الصحة النفسية
- **رعاية مخصصة:** توصيات مدعومة بالذكاء الاصطناعي وبرامج قابلة للتكيف
- **تجربة سلسة:** الرعاية تصل أينما ومتى احتجتها

## 🏥 لمقدمي الرعاية الصحية

- **نتائج أفضل:** تحسين الالتزام يؤدي إلى نجاح سريري
- **رؤى فورية:** بيانات مباشرة عن تقدم المرضى
- **تقليل الأعباء:** الدعم التلقائي يخفف العبء السريري

## 💊 للشركات الدوائية

- **الوصول للسوق:** إزالة العوائق وتوسيع قاعدة المرضى
- **ولاء العلامة:** بناء علاقات طويلة الأمد مع المرضى
- **توليد الأدلة الواقعية (RWE):** جمع بيانات قيّمة من العالم الحقيقي
- **ميزة تنظيمية:** إثبات القيمة للدافعين والجهات التنظيمية

## 🚀 التميز التنافسي

- **تكامل شامل:** المنصة الوحيدة التي تجمع الأعمدة الخمسة
- **التركيز على النتائج:** قياس وتحسين مدمجان في النظام
- **قابلية التوسع والنمطية:** نشر البرامج بشكل مستقل أو مدمج
- **مدعوم بالتكنولوجيا:** الذكاء الاصطناعي والتحليلات التنبؤية والأتمتة في الجوهر
